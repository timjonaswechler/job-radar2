# Effective Source Profile provenance

The active `compile_source` boundary records typed provenance alongside every successful `CompiledSource`. Profile-selected Sources receive one complete Effective Source Profile and provenance produced by the same deterministic specialization pipeline; Source-owned access remains a distinct branch and never receives a fabricated Effective Source Profile.

Provenance covers every execution-relevant terminal exactly once, uses final `detection`/`discovery`/`detail` vocabulary, and records Policy at its actual `policy.type` terminal. Origins are closed to Base Source Profile, Direct Source Fragment, and Source-owned access. Arrays remain atomic where ordering is semantic, dynamic maps use deterministic ordering, and metadata excluded from executable behavior is minimized.

Compiler rejection returns Diagnostics only and exposes no partial Effective Source Profile, Access Path, plan, or provenance. Source Live Check fingerprint preparation consumes the exact successful provenance; it never reconstructs provenance independently.
