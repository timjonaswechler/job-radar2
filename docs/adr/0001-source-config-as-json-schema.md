# Keep Source Config as profile-declared JSON separate from Source Overrides

Job Radar keeps each Source's stable access configuration as JSON because ATS and career-site configuration varies by profile and should not require database columns or migrations. Source Config contains only stable access values such as host, tenant, site, board slug, language, feed URL, or start URL; it must not contain search criteria such as keywords, roles, preferred locations, countries, radius, include rules, or exclude rules.

Source Profiles and their selected Access Paths declare the effective Source Config schema used for validation and UI/agent authoring. Source Overrides are a separate concept: they are controlled, compiler-validated behavior changes for one Source, not normal configuration and not a way to alter the Source Config contract. The final execution validity of a Source is determined by schema validation plus Profile Compiler validation of the selected Access Path, Source Config, and Source Overrides together.
