# Store source configuration as JSON validated by selected access-path schemas

Job Radar stores each source's stable access/configuration data as JSON and validates it against the source's selected access path: profile-level schema plus the chosen profile access-path schema, or the inline schema of a source-specific access path. Source configuration does not contain search criteria such as keywords, roles, locations, regions, or countries; those belong to search requests. This keeps source configuration extensible for agents and users while still giving the application a validation boundary; the main alternative, adapter-specific relational tables or columns, would make every new source or profile capability require a database migration.
