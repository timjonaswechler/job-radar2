# Triage Labels

Die Engineering-Skills verwenden fünf kanonische Triage-Rollen. Diese Tabelle ordnet sie den tatsächlichen Labels im GitHub-Repository zu.

| Kanonische Rolle | Label im Repository | Bedeutung |
|---|---|---|
| `needs-triage` | `needs-triage` | Maintainer muss das Issue bewerten |
| `needs-info` | `needs-info` | Wartet auf Informationen des Reporters |
| `ready-for-agent` | `ready-for-agent` | Vollständig spezifiziert und für einen AFK-Agent bereit |
| `ready-for-human` | `ready-for-human` | Benötigt menschliche Umsetzung |
| `wontfix` | `wontfix` | Wird nicht umgesetzt |

Wenn ein Skill eine kanonische Rolle nennt, wird das zugehörige Repository-Label aus dieser Tabelle verwendet.
